Inspired by Patrick Stolz's brilliant idea of mapping population exposure to travel, I decided to ask myself a question that only a data-loving introvert would: "How can I visit every country in the world while meeting the fewest number of people?"

This turned into a four-part journey through optimization problems, each more grounded in reality (and computational effort) than the last.

Case 1: All Grids, All Countries Per Grid (Simple Greedy)

In this beginner-level version, I pick the lowest population grid per country. If a grid belongs to multiple countries‚Äîgreat! One stop, many stamps. Teleportation assumed. Realism: zero. Population-avoidance: maximum.

Case 2: All Grids, One Country Per Grid (Strategic)

This one added some finesse. I enforced a one-to-one mapping: one grid per country. The problem then became an assignment challenge, solvable by Hungarian algorithms or similar combinatorial optimization techniques. It‚Äôs like speed dating with Earth‚Äôs least populated spots.

Case 3: Airports, Shared Country Grids (Greedy + TSP)

Now we fly. Using airport locations as nodes, I designed a greedy airport-to-airport TSP solver. After landing, I could claim any country sharing that airport grid, even if it required a hypothetical boat ride or gentle hike. This version introduced real-world logistics and a CO‚ÇÇ tracker to account for emissions.

Case 4: Airports, Host Country Only (Hardcore Realism)

This is the strictest and cleanest version. You can only count a country if you fly into an airport grid hosted by that country‚Äîno border-hopping. I computed the optimal airport sequence using TSP heuristics, minimized CO‚ÇÇ output, and ensured the population footprint was as low as possible.
