Many years ago I (probably like millions of people before!) had the idea to take the photo of a place over some time period. Now one can merge them in two ways, the first is a straight forward time-lapse and the second is to merge them into a single image with slices coming from each photo. However, like many of my ideas, it remained just that—an idea. I never got around to implementing it. But now, thanks to the super awesome LLMS, I’ve been accelerating my creative process and converting all these dormant ideas into reality with ease.

I restarted the effort for this earlier this month and when doing some internet-research, I came to know that there is a phrase for this called "High Dynamic Time Ranging (HDTR)" images. I was able to dig out the original image from which I got the actual idea from which is from Eirik Solheim’s year-long time-lapse project (One Year in One Image), where an entire year’s worth of photos is merged into a single image. The results look beautiful! I wish I had such a great sceneary to photograph.

My next task was to get hold of images of the same spot taken over some time. I dont have access to fancy scenary so I went with what I had which was the apartment balcony with my GoPro. While it technically worked, the results were not the most awe-inspiring.

While searching around for more, I stumbled upon this excellent HDTR project by Martin Krzywinski in this blog here, where he showed how to create insanely awesome images. Infact, most of the code for this project was based on his work. He has an excellent website with so many serious and fun projects. His blogs have become one of my always-open tabs in the last weeks.

Given that I did not have access to great set of photos, I looked to the internet and found another great website with nearly real-time photos of three locations in Darmstadt and a history of photos taken every 10 minutes. This was just the missing piece in my HDTR puzzle. Finally, piecing together all of the code and images together, I was able to create similar photos as Martin Krzywinski work (thought they dont have as smooth a transition as his. I was considering running an optimization algorithm with some sort of edge detection code to reduce these "bands").

The code itself was pretty straight forward, sort the images in forward time, take slices of width w/N where w is the width of each image and N is the total number of images, position the slices for each image sequentially. This gives us a first image which is quite good but still has jumps across the image boundary and shows bands as can be seen below.

These images were taken over a 24-hour period around the equinox last year to capture an equal balance of day and night. The final composition features four distinct directions of time movement. Each method provides a different way to visualize how time transitions throughout the day.

Now, I’m thinking of taking this idea a step further—what if I create an HDTR image spanning an entire year? Just like Solheim’s project, I could blend time over a full 12 months to create an even more immersive and intricate representation of change. This would capture not only daily variations but also the gradual shift of seasons, turning a single image into a powerful chronicle of time.

For those interested, I’ve made the code available on GitHub, so you can try it out yourself and experiment with different blending techniques.
